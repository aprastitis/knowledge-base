# Elon's 5-Step Process

## Type
Mental Model

## Summary
A framework for thinking clearly about and solving any problem by rigorously questioning every assumption before delegating, simplifying, accelerating, automating, and iterating in that specific order.

## The 5 Steps (in order)

1. **Question requirements** — Every requirement is a lie dressed up as a law. Ask: "Why must this be this way?" Don't accept "because we've always done it" or "because I said so."
2. **Delete** — Remove everything that isn't essential. If you can't describe what a feature does in one sentence, it's probably bloat. The goal is to have the fewest features possible that still solve the core problem.
3. **Simplify** — What remains should be as straightforward as possible. Design for the lowest common denominator of understanding. If it requires a manual, it's too complex.
4. **Accelerate** — Only speed up what remains. Don't optimize something that shouldn't exist in the first place.
5. **Automate** — Only automate steps 1–4. Automation applied to a broken process just makes the failure faster and more expensive.

## Why This Works
Most organizations do these in reverse: they automate first, then accelerate, then try to simplify something too complex to fix, and they're blocked by requirements they never questioned. Elon forces you to question before touching anything else.

## Application
- When starting a new project: question before planning
- When inheriting a system: delete before improving
- When facing resistance: "which step are we skipping?" is the diagnostic question
